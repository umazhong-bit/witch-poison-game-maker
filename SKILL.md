# 女巫的毒药游戏生成器

把老师的自然语言词表放入已经验证过的课堂玩法，而不是重新发明游戏规则。输出应稳定、离线可用，并保留教师临场判断“读对/读错”的权力。

## 生成流程

1. 从老师的话中提取英语单词或短语，去除重复项并保留首次出现顺序。
2. 明显且可确定的拼写错误可以修正，但要告诉老师；含义或拼写无法可靠判断时先询问。
3. 至少需要 4 个不同词项。少于 4 个时请老师补充；多于 25 个时，优先询问本轮使用哪些词，老师不介意时再随机选 25 个。
4. 使用 `scripts/generate_game.py` 生成 HTML。不要直接手写或重做模板。
5. 打开生成结果，确认棋盘、按钮和全屏功能正常，再交付文件路径。

## 固定教学设计

除非老师明确要求修改，否则始终采用以下规则：

- 5×5 棋盘，共 25 张卡；词数不足 25 时随机重复，尽量均匀覆盖全部词汇。
- 默认 A、B 两组，各自秘密选择 3 张毒药；另一组闭眼。两组可以选择相同卡片，确认时不得提示是否与另一组重合，避免泄露对方的毒药位置。
- 每组有 3 格药水生命，不累计分数；吃到一张毒药失去一格，先失去 3 格的一组失败。
- 若因两组毒药重合而在毒药全部吃完后仍无人失去 3 格，剩余生命多的一组获胜；生命相同则平局。
- 学生先点选一张卡并大声认读。读对时再次点击，才真正吃掉卡片。
- 翻开安全卡时，棋盘卡片和中央 `SAFE` 提示均使用鲜明绿色；毒药结果保留紫色，确保两种结果容易区分。
- 读错时教师点击“读错了，换一张”；原卡留在棋盘，本组必须换另一张继续这一轮。
- 不倒计时，不自动判定发音，不翻面；教师掌握课堂节奏。
- 默认面向教室横屏一体机。首屏使用“全屏并开始”，由老师点击后进入浏览器全屏；全屏时铺满 100vw × 100dvh，按 16:9 横屏布局保证棋盘、双方生命和教师控制区无需滚动即可同时显示。
- 提供显示/隐藏毒药、退出全屏和重新开始按钮；浏览器拒绝全屏时铺满当前窗口并提示老师按 F11。

## 命令

```bash
python3 scripts/generate_game.py \
  --words "museum, school, train station, shop, cinema, park, mountain, library" \
  --output "/目标目录/女巫的毒药.html"
```

脚本会同时生成一个 `.manifest.json` 文件，记录实际使用的词表和规则版本。最终 HTML 已嵌入全部样式、脚本和药水图片，双击即可离线使用。

## 不要做

- 不要为每次请求重新设计 UI 或玩法。
- 不要把“读错”理解为已经吃掉卡片。
- 不要擅自加入分数、倒计时、自动语音识别或学生个人设备模式。
- 不要引用外部图片、字体、CDN 或网络脚本。
